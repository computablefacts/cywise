<example>
    Voici un exemple du format attendu pour une vulnerabilite "Exposed .git Directory":

    ### Solution Simplifiee
    - Bloquer l'acces au repertoire .git via la configuration du serveur web
    - Verifier qu'aucun fichier sensible n'est accessible publiquement
    - Redemarrer le service web pour appliquer les changements
    - Tester l'acces depuis l'exterieur pour confirmer le blocage

    ### Solution Detaillee

    **Etape 1: Modifier la configuration Nginx**

    Editer le fichier de configuration du vhost (generalement dans /etc/nginx/sites-available/):

    ```nginx
    # AVANT (vulnerable)
    location / {{
        try_files $uri $uri/ =404;
    }}

    # APRES (securise)
    location / {{
        try_files $uri $uri/ =404;
    }}

    location ~ /\.git {{
        deny all;
        return 404;
    }}
    ```

    **Etape 2: Tester la configuration**
    ```bash
    sudo nginx -t
    ```

    **Etape 3: Appliquer les changements**
    ```bash
    sudo systemctl reload nginx
    ```

    **Impact:** Aucun downtime, rechargement a chaud de Nginx.

    **Bonne pratique:** Ne jamais deployer le dossier .git en production. Utiliser un processus de build/deploy qui exclut les fichiers de versioning.

    ### Risques si Non Corrige
    Un attaquant peut telecharger l'integralite du code source de l'application via le repertoire .git expose, revelant potentiellement des secrets (cles API, mots de passe) et la logique metier complete de l'application.

    ### Verification
    1. Tester l'acces: `curl -I http://{ip}:{port}/.git/config` (doit retourner 404)
       - Explique: Cette commande teste si le fichier est accessible. Un code 404 signifie que l'accès est bien bloqué.
    2. Verifier les logs Nginx: `tail -f /var/log/nginx/access.log` (acces bloques)
       - Explique: Vérifie que les tentatives d'accès sont bien refusées dans les logs du serveur.
    3. Scanner avec outil: `./gitdumper.sh http://{ip}:{port}/.git/` (doit echouer)
    4. **OBLIGATOIRE: Relancer un scan Cywise** pour confirmer que la vulnérabilité n'apparaît plus dans les résultats
</example>
